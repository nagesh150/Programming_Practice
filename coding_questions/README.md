## 🧠 How to Use This Guide

### Step 1: Understand the Problem

- Read the problem statement carefully
- Look at the visualization to understand what's happening

### Step 2: Trace Through Example

- Follow the step-by-step visual walkthrough
- Understand WHY each step happens

### Step 3: Code It Yourself

- Try to write the code WITHOUT looking at solution
- Then compare with provided solution

### Step 4: Analyze Complexity

- Understand time and space complexity
- Think about how it can be optimized

### When you see... → Think of...

| Problem Type              | Pattern                 |

| "Find pair with sum X"    | Two Pointers / Hash Map |
| "Subarray with condition" | Sliding Window          |

| "Matching brackets"       | Stack                   |
| "Level by level"          | BFS / Queue             |
| "All paths"               | DFS / Recursion         |
